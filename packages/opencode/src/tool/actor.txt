Manage subagents. Wrap arguments in `operation`, with `operation.action` selecting the command.

Actions:
- `run`: start a subagent and wait for the result.
- `spawn`: start a subagent in the background and return its `actor_id`.
- `status`: inspect a subagent state.
- `wait`: wait for a spawned subagent to finish.
- `cancel`: stop a running subagent.
- `send`: deliver a message to another actor inbox.

Use this for substantial delegated work such as parallel investigation, broad search, or independent review. Do not spawn for trivial single-file lookups.

Common subagent types:
- `explore`: read-only discovery and codebase search.
- `general`: multi-step research or execution.

Prompt the subagent with the goal, key context, and desired output format. `context` controls inheritance: `none` (default), `state`, or `full`.

`actor_id` resumes an existing actor session. `task_id` binds findings to an existing task from this session; do not invent task IDs.

Example:
{"operation":{"action":"run","subagent_type":"explore","description":"Find parser recovery","prompt":"Search parser-related files and report each recovery path with file:line."}}
