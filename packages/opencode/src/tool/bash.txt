Execute terminal commands in a persistent shell session.

Environment: OS `${os}`, shell `${shell}`.

Use this for terminal work such as git, npm, docker, tests, or `gh`. Prefer dedicated tools for file search, content search, file reads, and edits.

Key rules:
- Use `workdir` instead of `cd <dir> && ...`.
- Quote paths with spaces.
- Set `interactive: true` only when the command needs user input.
- If a command will create files or directories, verify the parent location first.
- Parallelize independent commands with separate tool calls; use `${chaining}` only when steps depend on each other.
- Avoid newline-separated command lists.

Git safety:
- Only commit or push when the user explicitly asks.
- Never change git config, skip hooks, or use destructive git commands unless explicitly requested.
- Avoid interactive git commands.

If output exceeds ${maxLines} lines or ${maxBytes} bytes, it will be truncated and the full output will be written to a file for follow-up reading.
