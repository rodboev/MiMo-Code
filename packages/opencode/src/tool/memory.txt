Search stored memory with BM25 across session, project, global, or optional `cc` scopes.

Use short, distinctive queries such as a task ID, function name, flag, or rare phrase. Queries are OR-ranked, so a few specific terms work better than a long generic phrase.

Guidance:
- A search hit is authoritative even if another phrasing returned nothing.
- If you need the exact literal form of a path, URL, token, port, or command and memory only gives a paraphrase, use `history`.
- If search returns 0, retry with fewer or rarer terms, then widen scope.

After search, use `Read` to inspect the full memory file.
